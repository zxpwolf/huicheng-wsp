需求拆解 - 用户故事

核心用户场景与验收标准

👨‍⚕️ 医生/健康管理师
作为医生，我希望系统能根据用户健康数据自动生成标准化报告，这样我可以快速了解用户状况并给出专业建议。
验收标准：输入 JSON → 30秒内输出完整 PDF 报告，包含所有必填字段
🏥 医疗机构
作为机构运营方，我们需要批量处理用户报告，系统应支持高并发和自动化流程。
验收标准：支持 API 调用，单次处理 < 5秒，日处理量 > 10,000 份
👤 减重用户
作为减重用户，我希望获得清晰易懂的健康报告，并能通过 AI 获取个性化的减重建议。
验收标准：报告包含可视化图表，AI 解读通俗易懂且可操作
🔧 系统管理员
作为管理员，我需要能够配置报告模板、调整分级标准，并监控系统运行状态。
验收标准：模板可配置，分级规则可热更新，有完整的日志和监控

功能清单

系统核心功能模块

📥 输入处理

  • JSON 数据解析
    解析 record + derived 结构化数据
  • 数据校验
    必填项检查、格式验证、范围校验
  • 数据补全
    缺失字段标记、默认值填充

⚙️ 核心处理

  • 自动分级计算
    BMI、肥胖分度、代谢综合征判断
  • 模板填充
    V17 模板变量替换、条件渲染
  • 闸门控制
    多级校验、质量评分、异常拦截

📤 输出生成

  • PDF 报告生成
    WeasyPrint 渲染、分页、水印
  • MD 文件生成
    结构化 Markdown、AI 提示词模板
  • 文件存储
    OSS 上传、CDN 分发、生命周期管理

🔧 辅助功能

  • 日志记录
    操作日志、错误追踪、审计记录
  • 监控告警
    性能监控、异常告警、健康检查
  • 配置管理
    模板配置、规则配置、环境配置
