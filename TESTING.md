# 荟城减重 WSP 系统 - 测试指南

## 端到端测试套件

本项目包含完整的端到端测试脚本,用于验证系统的各项功能。

### 运行测试

```bash
# 激活虚拟环境
source .venv/bin/activate

# 安装依赖(首次运行)
pip install requests

# 运行测试
python test_e2e.py
```

### 测试覆盖范围

#### ✅ 测试1: 仪表盘数据
- 验证API返回数据结构完整性
- 检查趋势数据是否按日期排序
- 验证趋势数据包含完整的7天
- 确认每个数据点都有count字段

#### ✅ 测试2: 创建报告 - 正常数据
- 提交完整的报告数据
- 验证报告成功创建
- 检查返回的报告ID和状态

#### ✅ 测试3: 创建报告 - 边界值验证
测试17个边界场景:
- 身高: 最小值(100cm)、最大值(250cm)、超限(251cm)
- 体重: 最小值(30kg)、最大值(300kg)、超限(301kg)
- 收缩压: 边界值(250mmHg)、超限(251mmHg)
- 舒张压: 边界值(150mmHg)、超限(151mmHg)
- 心率: 低边界(40次/分)、高边界(200次/分)、超限(201次/分)
- 体脂率: 边界值(5%)、超限(61%)
- 内脏脂肪等级: 边界值(1级)、超限(60级)

#### ✅ 测试4: 字符串数值转换
- 验证前端传来的字符串类型数值能正确转换
- 测试整数字符串('170')和浮点数字符串('22.5')
- 确保转换后能正常进行数值比较

#### ✅ 测试5: 报告生成
- 测试PDF内容生成
- 测试MD内容生成
- 验证报告详情可正常获取

#### ✅ 测试6: 闸门逻辑
- 测试红旗项阻断机制
- 验证闸门日志正确记录
- 检查阻断原因是否准确

#### ✅ 测试7: 报告列表和搜索
- 获取全部报告列表
- 关键词搜索功能
- 按状态过滤(completed/blocked)

### 开发流程建议

**每次修改代码后都应该运行测试:**

1. **修改前**: 先运行测试,确保当前代码正常
2. **修改后**: 再次运行测试,验证修改没有引入新问题
3. **提交前**: 确保所有测试通过

### 添加新测试

如需添加新的测试场景,请遵循以下原则:

1. **命名规范**: 函数名以`test_`开头,清晰描述测试内容
2. **日志输出**: 使用`log()`函数输出测试结果
3. **返回值**: 返回布尔值表示测试是否通过
4. **异常处理**: 捕获并记录所有异常
5. **注册测试**: 在`run_all_tests()`中添加新测试

### 持续集成建议

可以将此测试脚本集成到CI/CD流程中:

```yaml
# 示例: GitHub Actions
name: Run Tests
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Set up Python
        uses: actions/setup-python@v2
        with:
          python-version: '3.14'
      - name: Install dependencies
        run: |
          pip install -r requirements.txt
          pip install requests
      - name: Run tests
        run: python test_e2e.py
```

### 注意事项

⚠️ **重要提醒:**
- 测试会向数据库写入测试数据,建议在测试环境中运行
- 确保Flask应用正在运行(`python app.py`)
- 测试需要网络连接(访问http://127.0.0.1:5000)
- 如果测试失败,请查看详细错误信息并修复问题
